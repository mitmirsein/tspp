## 0. 한 줄 요약

매주 반복되는 설교 준비의 부담을 덜되, **본문 충실·회중 적용·목회적 권위**를 침범하지 않는 보조 도구. TAWP의 학술 글쓰기 자동화 원칙(*형식≠품질·HITL 책임귀속·환각 방지*)을 설교 도메인에 이전한다. 최종 산출은 **간결한 설교 개요(1~3쪽, 짧은 해설 동반)**.

## 1. 무엇이 동일하고 무엇이 다른가 (TAWP vs TSPP)

| 측면 | TAWP (학술) | TSPP (설교) |
|---|---|---|
| 1차 독자 | 학계 동료·심사위원 | 회중 (특정 교회·연령·맥락) |
| 출처 인용 | 엄격 (DOI/SBL 각주) | 설교 내부에는 통상 미인용; 설교자가 *참고*만 |
| 핵심 가치 | 학술 무결성·논증력 | 본문 충실 + 회중 적용 + 영적 권위 |
| 길이 | 3,000 어절 (publication 기본) | 1~3쪽 개요 (또는 30분 구두 원고) |
| 시간 사이클 | 1편/주제당 수 주 | 매주 반복 (절기·시리즈) |
| Hard gate | 형식·정합·바인딩 | 본문 오용·적용 비약·균형·전달성 |
| 최종 산출 | Markdown essay | 설교 개요 (+ 선택적 전체 원고) |
| 보이스 | 주제별 register (L2 단층) | **설교자 개인 동일성(L1-개인) + 상황 register(L2)** — 이중 축 (→ §4-5) |
| 책임 귀속 | 저자 (논문) | **목회자 (설교의 영적 책임)** |

## 2.5. ⚡ 결정적 발견 — 호밀레틱 *엔진*은 이미 존재한다 (Theology_Ontology_Starter)

TSPP를 "처음부터" 짓는다는 전제는 틀렸다. 사용자의 또 다른 프로젝트 **`Theology_Ontology_Starter`(TOS, v5.8)**가 이미 성숙한 *온톨로지 기반 설교 작성 엔진*을 갖추고 있다(git·pytest 113+·`uv` CLI·`manager.py`).

**TOS가 이미 가진 것** (`frameworks/`):
- **The Decalogue** — 10종 설교 프레임워크(`homiletics.md`): ① Hook-Hurdle ② Narrative Arc ③ Idol Deconstruction ④ Polyphonic ⑤ 4P+Courtroom ⑥ Incarnational Descent ⑦ Aporia Meditation ⑧ Lament ⑨ Prophetic Imagination ⑩ Sacramental
- **Meta-Compositor**(`homiletics_compositor.md`) — 온톨로지 신호(aporia·entity·claim)로 Primary+Secondary 프레임 자동 조합 + 호환성 매트릭스(시너지/충돌/조건부) + Fusion Blueprint
- **품질 게이트·4-Pass·루브릭** — §1.2 Grounding/신학정합/Aporia처리, Parse→Draft→Audit→Rewrite, 5차원 채점, 실패 정책(§1.5)·안전 가드레일(§1.7)·`QUALITY_GATES.md`·`ROUTING_MATRIX.md`
- **청중 적응 레이어**(§1.9) — adult / young_adult / youth
- **다운스트림** — `slides_core`·`digital_shortform`·`education`·`bible_study`·`deep_qt`·`card_news`
- **Homiletic Architect 페르소나 + 톤 스펙**(§1) — "C.S.루이스 지성 + 유진 피터슨 목회 감수성 + 팀 켈러 변증, *번역투 아닌 한국 강단 구어체*"

→ 즉 CONCEPT v0.1~v0.2가 "신규 재설계"로 잡았던 **outline-builder·application-mapper·homiletic-redteam·delivery-adapter·청중 적응**은 *상당 부분 TOS에 이미 있다.* 새로 짓는 건 중복이다.

### 3-프로젝트 분업 (재정의의 핵심)

| 프로젝트 | 강점 | 설교 준비에서의 역할 |
|---|---|---|
| **TAWP** | 주해·학술 리서치 정밀(fan-out 5엔진·binding·claim-ledger·유령인용 차단) | 본문 주해의 *근거*를 캔다 → TOS 온톨로지 입력 품질↑ |
| **TOS** | 호밀레틱 *작성* 엔진(Decalogue·compositor·게이트·청중·다운스트림) | 온톨로지 → 설교 구조·초안·미디어 |
| **TSPP** ★ | (신규) *오케스트레이션* + **설교자 보이스 아키텍처** + 목회 윤리 | 둘을 잇고, 둘 다 없는 것을 채운다 |

### TSPP가 *진짜로* 새로 더하는 것 (두 상류에 없는 것)

1. **TAWP ↔ TOS 브리지** — 주해 리서치(TAWP fan-out) → 온톨로지(TOS 입력, §1.8 Textual Deep Dive를 *근거로 강화*) → Decalogue 작성. 두 엔진의 데이터 계약을 잇는 오케스트레이션.
2. **설교자 보이스 아키텍처**(§4-5) — TOS의 톤 스펙(§1)은 *고정 페르소나 한 겹*(루이스+피터슨+켈러)이다. TSPP는 그 위에 **L1-개인(설교자 동일성, opt-in) + L2-상황(본문장르×청중×절기×tier)** 2층을 얹어 "그 목회자 *자신의* 목소리"를 보존한다. TOS의 "번역투 금지"는 우리가 TAWP에 막 구현한 한국어 산문 통사 신호(L1-B)와 같은 축 — 공유 가능.
3. **목회 윤리 헌법**(§8) — 책임 귀속·대필 방지·HITL sign-off. TOS는 안전 가드레일(§1.7)은 있으나 *설교의 영적 책임 귀속*을 명문화한 헌법 층은 TSPP가 더한다.

> **재정의 요약**: TSPP ≠ "설교 파이프라인 신축". TSPP = **TAWP(리서치) + TOS(작성)를 잇고 그 위에 보이스·윤리 층을 얹는 오케스트레이션 프로젝트.** 이로써 §3(재설계 부품)은 대폭 축소되고, F5(부품 공유)는 "두 상류와 어떻게 결합하느냐"로 확장된다.
>
> **⚠️ 전제: TSPP는 단독 레포·단독 프로젝트다.** TAWP/TOS 안에 nesting하지 않는다(`projects/tawp/sermon/` 같은 sub-project 금지). 따라서 TAWP·TOS는 TSPP의 *상류 의존*이지 *부모*가 아니며, 결합은 **명시적 인터페이스**(데이터 계약 / 산출물 소비 / 선택적 복사·submodule)로만 이뤄진다. 세 레포는 독립적으로 버전·배포된다. ← 이 전제가 F5 선택지를 좁힌다.

## 4. 신규 *핵심 가치* (TAWP에 없는 것, 설교에 결정적)

### 4-1. 본문 오용(eisegesis) 차단 — **결정적**
TAWP의 `binding_verifier`가 *학술 인용 적정성*을 다룬다면, TSPP는 **"본문이 그 메시지를 정말 말하는가"**를 다룬다. 자기 확증 편향·proof-texting·주제로 본문을 끌어다 쓰기를 잡는 *최강* 게이트. → TAWP의 binding 가치가 *설교에서 훨씬 critical*.

### 4-2. 회중 맥락 입력 — *새 1차 reference*
TAWP의 `essay_terms`가 *주제별*이라면, TSPP는 *회중별*이 추가됨. 같은 본문(예: 마 25:14-30 달란트 비유)도 청년부·노년·정치사회 위기 시기에 *다른 적용*. 회중 프로파일이 1차 reference.

**회중 컨텍스트의 차원** ★ (사용자 요청 — "교회 청중 분위기" 반영) — `audience_profile.json`:
- *인구학*: 연령 분포·세대 구성 (TOS 청중 모드 adult/young_adult/youth를 세분)
- *신앙 색채*: 신앙 성숙도·교파 전통(장로/감리/순복음…)·신학 보수–진보 스펙트럼
- *정서·분위기*: 회중의 현재 정서(소진·갈등·부흥·상실)·예배 분위기(정적/역동)
- *상황*: 지역(도시/농어촌)·사회 사건(재난·선거·경제)·교회 내 사건(분쟁·이전·부흥회)
- *예전 위치*: 절기(대림·사순·부활·성령강림)·시리즈 내 위치

이 프로파일은 **두 곳에 동시에** 먹는다 — ① **L2-상황 보이스**(§4-5)의 톤 결정(같은 본문도 소진된 회중엔 목회적, 안일한 회중엔 예언적), ② **적용 매핑**(§4-3)의 "오늘 이 회중에게". TOS 청중 적응 레이어(§1.9, 연령 3모드)를 *입력으로 강화*한다(정서·상황·예전까지 수용). **단, 헌법(§8): 회중 프로파일은 *목회자가 작성*하지 AI가 추정하지 않는다.**

### 4-3. 본문→오늘 매핑
학술은 *명제 논증*, 설교는 *시대 적용*. 별도 `application-mapper`가 본문의 신학적 핵심을 *오늘의 회중에게 무슨 말씀인가*로 변환. 단, 적용이 본문을 *왜곡*하면 4-1이 차단.

### 4-4. 전달 어댑테이션
학술은 *읽는* 글, 설교는 *듣는* 말. 호흡·반복·강조·구두 친화 어휘. 같은 메시지를 10분/30분/45분 시간에 맞게 길이 조정 (TAWP `manuscript-expander` 변형).

### 4-5. 설교자 보이스 — 개인 동일성 + 상황 register ★ (TAWP 보이스 구조의 *장르 이식*)

TAWP는 "라이팅 품질은 사후 linter가 아니라 *상류 보이스 세팅*에서 나온다"는 원칙으로 **보이스 2층 아키텍처**(L1 보편 헌장 + L2 per-essay 팔레트 + audit 계기판)를 정립했다. 이 *패턴*은 설교에 이식할 가치가 크지만, **두 가지 장르적 전도(inversion)** 없이 복사하면 오작동한다. (이것이 TSPP를 TAWP "고도화"가 아니라 *자매 프로젝트*로 분리하는 핵심 근거이기도 하다.)

**(1) L1은 복사가 아니라 *전도*다.** 학술 L1("사유 안에 거주하는 저자")이 엄격히 *금지*하는 것을 설교는 오히려 *요구*한다:

| 축 | 학술 L1 (거주하는 저자) | 설교자 L1 (회중 앞에 선 설교자) |
|---|---|---|
| 청중 | 2인칭 호명 금지(§8 심사평) | 직접 호명이 생명 |
| 결론 | 선고 금지·명제만 | 케리그마적 선포·결단의 부름 |
| 당위 | 금지(§8) | 권면·적용의 당위가 정당 |
| 시점 동선 | 사유 안에 머묾 | 본문 → 회중 → 응답으로 이동 |

→ 그러므로 TSPP는 **별도의 호밀레틱 L1 헌장**("본문 안에 거주하면서 회중 앞에 선 설교자")을 가진다. 또한 TAWP의 `tawp_audit §8`(학술 시점 검출기)을 *그대로 재사용하면 안 된다* — 정당한 설교 어법(직접 호명·선포·권면)을 전부 오검출한다. TSPP는 **신호가 다른(일부는 반대인) 호밀레틱 보이스 계기판**을 새로 만든다.

**(2) 보이스 축이 *이중*이다.** 학술 보이스는 *주제별 register*(L2 팔레트) 단층이었지만, 설교는 한 층이 더 있다:

- **L1-개인 (설교자 동일성)** — 학술과 결정적으로 다른 지점. 한 설교자는 모든 설교에 걸쳐 *지속되는 고유 목소리*(어휘·리듬·신학적 강조점·목회적 성정)를 가진다. §13의 "개인 `style.json`" 발상을 **1급 레이어로 승격**한 것. **opt-in** — 목회자가 자기 보이스를 등록하며, 미등록이면 호밀레틱 L1 보편 헌장만 적용(정상 degraded). 산출 `preacher_voice.json`.
- **L2-상황 (본문·청중·절기 register)** — 설교마다 달라지는 톤. 본문 장르(내러티브 / 지혜 / 서신 / 예언·묵시) × 청중(`audience_profile`) × 절기(`lectionary`) × tier(강해 / 예언 / 목회 / 전도 / 교리)로 결정. 학술 `voice_palette.json`에 대응하는 **호밀레틱 팔레트**(`homiletic_voice_palette.json`).

`personas/Preacher_*.md` 5종은 L2 톤의 *어조 차용원*(견해 인용이 아님)으로, `Audience_Voice.md`는 L2의 청중 시뮬레이션 입력으로 연결된다. 보이스는 TAWP와 동일하게 **개요 작성 *전*(Phase 5 진입 전)에 고정**한다 — 계기판은 그 결과를 점검하는 안전망이지 원인이 아니다.

**(3) 윤리적 정합 — 개인 보이스 보존이 곧 *대필 방지*다.** 이 설계는 헌법(§8 "AI는 설교를 대신 쓰지 않는다")과 직결된다. AI가 *설교자 자신의 목소리*를 보존·증폭하도록 강제하면, 산출은 "AI가 쓴 매끈한 남의 글"이 아니라 "그 목회자가 더 깊이 준비한 *자기* 설교"가 된다. **L1-개인은 편의 기능이 아니라 안전장치다.**

**(4) 보이스 인제스트 — 설교문 샘플에서 L1-개인 *추출* ★ (사용자 요청)**

L1-개인을 *백지에서 작성*하는 대신, **설교자가 자기 설교문 샘플 N편(권장 5~10편)을 입력하면 시스템이 개인 어투·문체를 프로파일링**해 `preacher_voice.json` 초안을 자동 생성한다(HITL 승인 후 확정). 프로파일링 대상:

- **어휘대(lexicon)** — 자주 쓰는 신학 어휘·구어 표현·관용구, 회피 경향 어휘
- **리듬(cadence)** — 문장 길이 분포, 짧은 단언 vs 긴 전개 비율, 반복·점층 습관
- **수사 습관** — 질문 던지기·예화 유형·성구 인용 빈도와 방식
- **신학 강조점** — 반복 등장하는 주제·무게중심(은혜/십자가/언약 등)
- **구어체 표지** — 강단 화법(청중 호명 방식·권면 어조·결단 초청 패턴)

→ 산출은 *그 목회자의 목소리 지문*이고, 이후 모든 설교의 L1-개인으로 주입된다. **이것이 (3)의 대필 방지를 *실물로* 구현하는 메커니즘이다** — AI가 남의 매끈한 문체를 입히는 게 아니라 *설교자 자신의 문체를 학습해 되돌려준다.*

- *정직성*: 추출 프로파일은 *초안*이며 HITL이 교정·승인한다(자동 확정 아님). 샘플 부족 시 채워진 차원만 쓰고 나머지는 호밀레틱 L1 보편 헌장으로 폴백.
- *프라이버시*: 설교문 샘플은 사용자 자산으로 **로컬 처리**, 외부 전송·학습 데이터화 금지(헌법 §회중 익명성과 같은 결).
- *드리프트 점검*: 산출 설교가 개인 보이스에서 벗어나면 호밀레틱 보이스 계기판이 신호(형식 점수 아님, HITL worklist).
- *기술 현실*: 문체 프로파일링은 LLM 분석 과제로 즉시 PoC 가능. 정확도 검증은 추출 프로파일로 생성한 설교를 원 샘플과 대조하는 HITL 평가로(§13).

## 5. 설교 준비 8-Phase 흐름 (제안)

```
[입력: 본문 / 절기 / 회중 프로파일]
   │
 Phase 1: 본문 선택      ──── lectionary / 강해 시리즈 / 시즈널 (선택)
   │
 Phase 2: 본문 묵상      ──── sermon-mentor: 묵상 씨앗 추출 (HITL 승인)
   │
 Phase 3: 주해            ──── exegesis-toolkit: 원어·문맥·교차 본문
   │
 Phase 4: 자료 fan-out   ──── 주석서·관련 신학 논문·교부 주해 (research_fanout 재사용)
   │
 Phase 5: 메시지 + 개요  ──── outline-builder + Foundation Gate
   │                            (본문 충실 + 적용 정합 + 회중 alignment)
 Phase 6: 적대적 레드팀  ──── homiletic-redteam: eisegesis·자기 확증·균형
   │                            (HITL 승인)
 Phase 7: 개요/원고 작성 ──── 간결한 개요 (기본) + 선택적 분량 확장 (manuscript-expander)
   │
 Phase 8: 최종 감사       ──── binding_verifier (본문 정합) + audience_alignment
   │                            + 전달 검수 (HITL 최종 sign-off)
   ▼
[최종: 설교 개요 + (선택) 전체 원고 + 보조 자료]
```

> 모든 Phase 사이에 **HITL 결정 카드** — TAWP와 동일 (✅승인 / ↻돌아가기 / ✏️수정 / ⏸보류).
>
> **보이스 고정 시점**: 설교자 보이스(L1-개인 + L2-상황, §4-5)는 **Phase 5(개요) 진입 *전*에 세팅**한다 — 청중·절기·tier가 정해지는 Phase 1~2에서 L2를 고르고, 등록된 `preacher_voice.json`이 있으면 L1-개인을 얹는다. Phase 8 호밀레틱 계기판은 그 흔들림을 점검하는 안전망이다.

## 8. 목회 윤리 헌법 초안 (CLAUDE/AGENTS/GEMINI.md에 박을 핵심)

- **영적 권위·책임은 목회자에게 귀속.** AI는 sign-off 절대 금지. *모든* 단계 최종 결정은 목회자.
- **AI는 설교를 *대신 쓰지 않음*, *준비 도구*임.** MANUAL 첫 줄에 명시.
- **본문 충실 > 시대 적합성.** 적용이 본문을 *왜곡*하면 차단. eisegesis 자동 검출 게이트가 hard gate.
- **묵상 씨앗 존중** (TAWP 메모리와 동일 철학). 설교자의 영적 통찰이 1차 source — AI가 묵상을 *대체*하지 않음.
- **회중 맥락은 사용자 입력.** AI가 회중을 *추정*하지 않음. audience_profile은 *목회자가 작성*.
- **신학적 분별 강제.** 이단·오해·일방 편향·proof-texting을 자동 검출하여 worklist로. (judgment는 목회자.)
- **출처 신중성.** EvidencePack의 모든 자료는 *참고*일 뿐, 본문 텍스트의 권위를 대체하지 못함.
- **회중 익명성·사례 신중.** 회중 안 일화를 예화로 쓸 때는 *동의·익명화* 필요(목회 윤리).
- **차별·정치 편향 차단.** 적용·예화에서 특정 정치·인종·계층 차별성을 redteam이 차단.

## 10. TAWP 철학의 설교 도메인 전이 — *왜 의미 있는가*

TAWP가 *학술 자동화의 정직한 한계 인정*("형식≠품질", "binding은 HITL 책임", "묵상 씨앗 존중")이라는 원칙을 정립했다. 이 원칙이 **설교 도메인에서 더 절실**하다 — 설교는 본질적으로 *목회자의 영적 권위*가 핵심인데, 그 영역을 AI가 침범하면 안 된다.

- TAWP에서: 의미 검증 자동화 한계는 *학술 정직성* 문제
- TSPP에서: 의미 검증 자동화 한계는 *영적 책임* 문제 — *훨씬 무거움*

→ TSPP는 *TAWP의 윤리 원칙을 강화 적용*하는 사례가 된다.

## 11. 다음 단계 제안 (실행 시)

### 옵션 A. 개념 PoC (가장 가벼움)
한 본문(예: 마 5:1-12 팔복)으로 *수동 워크플로* 한 사이클 — TAWP 리서치 → (수동) 온톨로지 → TOS Decalogue 작성을 손으로 이어 *어디가 매끄럽고 어디가 끊기는지* 확인. 새 부품 없이 시작.

### 옵션 B. 단독 레포 골격 신설
**`tspp` 단독 git 레포** + 헌법 3파일 + MANUAL.md + `references/homiletic-voice.md`(L1) + 보이스 데이터 2종 + 첫 스킬 sermon-mentor. TAWP·TOS는 *상류 의존*으로만 연결(F5). 1주차 PoC 가능 상태.

### 옵션 C. **TAWP↔TOS 브리지 어댑터** (이 프로젝트의 심장)
TAWP EvidencePack/주해 근거 → TOS 온톨로지 입력(entities/claims/aporia JSONL) 변환 `scripts/bridge_*.py`. 두 상류의 데이터 계약을 잇는 최소 어댑터. 이게 되면 "리서치→작성" 자동 연결의 뼈대 완성.

### 옵션 D. 한 본문으로 *수직* PoC
한 본문 + 한 회중 프로파일(TOS 청중 모드) + 한 Decalogue 프레임으로 전체 흐름 한 번 돌려 *어디서 막히고 어디가 매끄러운지* 발견. 보이스 세팅(§4-5) 적용 여부 A/B 비교 권장.

- **git submodule** (TAWP/TOS를 서브모듈로; 업데이트 동기화 정합, 빌드 복잡)
- ~~TAWP/TOS 안에 sub-project~~ — **제외**(TSPP는 단독 레포·프로젝트, §2.5 전제)
- 혼합: TAWP는 산출물 소비, TOS는 submodule 등 상류별 차등도 가능

## 13. 미해결·심화 논의 거리 (브레인스토밍용)

- **AI 대필 설교의 윤리적 경계** — 어디까지가 *준비 보조*이고 어디부터가 *대필*인가?
- **회중 다양성** — 한 설교가 모든 회중을 다 만족시킬 수 없음 → audience_profile의 *우선순위* 입력 방식

- **장기 시리즈 일관성** — 시리즈 1~12편의 흐름·반복·발전을 어떻게 추적할까 (run_manifest 확장?)
- **설교 후 피드백 루프** — 회중 반응·자기 평가가 다음 설교 준비에 어떻게 반영될까 (선택 단계)
- **다국어 회중** — 한국어 설교 + 영어 자막·요약 동시 산출 가능성
- **시청각 보조** — 슬라이드·찬송·영상 연결 (범위 밖? 또는 별도 모듈?)
- **신학 자료 한계** — 한국 신학 출판물의 디지털화 정도·KCI 커버리지·교부 한국어 번역 가용성

## 부록: 디렉터리 구조 초안 (PoC 시작 시)

> **단독 레포 전제(§2.5)** 반영 — TAWP·TOS는 트리 *밖* 상류 의존이고, TSPP는 브리지·보이스·윤리에 집중한다. 신축은 ★표만.

```
tspp/                              # 단독 git 레포 (TAWP·TOS 트리 밖)
├── CONCEPT.md                     # 이 문서
├── MANUAL.md                      # (PoC 후) 목회자용 사용 매뉴얼
├── README.md                      # (PoC 후) 기술 가이드
├── CLAUDE.md / AGENTS.md / GEMINI.md   # (PoC 후) 목회 윤리 헌법 (§8)
├── references/
│   └── homiletic-voice.md     ★   # 설교자 L1 보편 헌장 (§4-5; 학술 authorial-voice.md의 *전도*판)
├── data/
│   ├── homiletic_voice_palette.json ★  # L2-상황 톤 팔레트 (본문장르×청중×절기×tier)
│   ├── preacher_voice.example.json  ★  # L1-개인 보이스 카드 (opt-in)
│   └── lectionary.json        ★   # (선택) 교회력·시리즈
├── input/                         # 목회자 제공 (로컬·비공개)
│   ├── sermon_samples/        ★   # 개인 보이스 인제스트용 설교문 N편 (§4-5 (4))
│   └── audience_profile.json  ★   # 회중 컨텍스트 (연령·정서·상황·예전, §4-2)
├── scripts/
│   ├── bridge_evidence_to_ontology.py ★  # TAWP EvidencePack → TOS 온톨로지 JSONL (핵심)
│   ├── voice_ingest.py        ★   # 설교문 샘플 → preacher_voice.json 프로파일 추출 (§4-5 (4))
│   ├── voice_resolve.py       ★   # L1-개인 + L2-상황 해석·주입
│   └── pastoral_gate.py       ★   # 책임귀속·대필방지 HITL 체크
├── skills/
│   └── sermon-mentor/         ★   # 본문 묵상 → 메시지 + 보이스 세팅 (TAWP research-mentor 변형)
└── (상류 의존 — F5로 결합, 트리 밖)
    ├── ↑ TAWP   : research_fanout·binding_verifier·claim_ledger (주해 리서치)
    └── ↑ TOS    : homiletics(Decalogue)·compositor·게이트·청중·slides/shortform (설교 작성)
```
