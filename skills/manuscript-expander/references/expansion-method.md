# 설교 원고 확장 방법론 (Expansion Method)

> `manuscript-expander` 스킬의 방법론 본체. 개요(뼈대) → 전체 구어 원고(살).
> 설계: 보이스 고정 원칙 Phase 7·§6. 상류: sermon-outline(개요)·voice(3층)·sermon-mentor(씨앗).

---

## 1. 확장의 정의 — 펴는 것이지 새로 쓰는 것이 아니다

전체 원고는 개요의 **논리·구조·보이스를 구어로 펴는** 작업이다. 개요에 *없는* 메시지·논점·예화를 새로 끼워넣는 것이 아니다. 개요가 뼈대라면 원고는 그 뼈대에 *그 설교자의* 살을 붙인 것이다 — 살이 남의 것이면 대필이다.

**왜 전체 원고가 선택(★ 아님)인가**(개요 기본 산출 원칙): AI가 쓸수록 대필 위험이 커진다. 개요로 충분한 설교자에겐 권하지 않는다. 전체 원고가 필요한 경우(원고 설교 전통·번역·아카이브)에만, 그것도 *개요를 편* 형태로.

---

## 2. 정직성의 division (파이프라인 일관)

```
[게이트]   sermon_outline.md frontmatter approved:true  (개요 HITL 통과)
[확장]     이 스킬(에이전트)        → full_manuscript.md   (구어 산문)
[전달]     scripts/delivery_pack.py → 시간·섹션·낭독 보조 → delivery_pack.md(에이전트)
[점검]     scripts/homiletic_audit.py   → 비점수 worklist(구어 산문이라 정밀)
```

스크립트는 시간·구조를 잰다. 원고(산문)는 에이전트가 쓰되 §3 규율에 정착한다. 점검은 확장 *후* 계기판으로 — 전체 원고는 구어 산문이라 hapnida 아티팩트 없이 정확하다.

---

## 3. 섹션별 펴기 (개요 → 원고)

개요의 각 섹션을 다음 원칙으로 편다. 본문장르(L2)에 따라 변주.

| 개요 섹션 | 펴는 법 | 경계 |
|---|---|---|
| **도입** | 회중을 본문 장면 안으로 — 거주·전향(L1 §2). 직접 호명으로 연다. | 배경 강의로 시작하지 않음(강사화) |
| **논점 3–5** | 각 논점을 본문에 머물러 풀되, 회중의 오늘로 돌아섬. supporting_refs는 소화해 비추되 학자명 나열 금지 | 새 논점 추가 금지 |
| **예화** | 개요가 지정한 예화를 구체적 장면으로. 본문장르·persona 어조 | 새 일화 지어내기 금지(유령) |
| **적용** | 본문→오늘 이 회중(audience 음높이). 본문이 요구하는 곳으로 | eisegesis — 본문 미끼 금지 |
| **긴장 보존** | disposition 따라 살림. 봉합·강제개방 금지 | 평탄화 금지 |
| **결단·부름** | tier에 맞게(전도=초청·제조금지 / 예언='우리' / 목회=동행) | 감정 제조 금지(선동) |
| **기도·마무리** | 회중을 대표한 1인칭 복수 기도, 짧고 직접 | 상투구 기도 금지 |

## 4. 구어 register — 원고가 *말*이 되게

전체 원고는 *읽을 글*이 아니라 *말할 말*이다.

- **직접 호명**(여러분·형제자매)이 살아 있어야 한다 — L1 §2의 생명. 계기판은 그 *결핍*을 경보한다.
- **'~습니다'체 구어** 중심. 문어 '~다' 평서·논문체(되어진다·다름 아니다·본고)로 굳지 않게.
- **듣기 좋은 호흡** — 긴 전개 뒤 짧은 단언으로 못 박기. 한 호흡에 한 생각.
- **시그니처 표현 살림** — preacher_voice 시그니처가 원고에 등장하게(드리프트 점검은 그 소실을 본다).
- **상투구 금지** — "은혜가 되시기 바랍니다" 류 죽은 관용구로 분량을 늘리지 않는다.

## 5. 분량과 전달 (Phase 3 — Delivery Pack)

`delivery_pack.py`가 측정하고, 에이전트가 `delivery_pack.md`로 정리한다.

- **시간**: 전체 + 섹션별 추정(글자수÷분당글자수, **대략**). 한 섹션이 40% 넘으면 배분 점검.
- **over**: 군더더기·반복·곁가지 절제. 분량이 아니라 *밀도*를 줄인다.
- **under**: 전개·예화 보강하되 *개요에 있는* 것을 더 충실히. 분량 채우려 새 내용·반복·상투구 금지(계기판이 잡는다).
- **낭독 보조**: 원어(예: πτωχός)는 강단에서 음역보다 *풀어 말하기* 권장 · 숫자·성구는 소리 낼 형태로(5:3 → "다섯 장 세 절") · 120자 넘는 긴 문장은 끊어/나눠 읽기.
- 추정·발음은 설교자가 자기 속도·회중에 맞춰 최종 보정(녹음 대조 권장).

---

## 6. 게이트와 점검 (Phase 0·4·5)

- **Phase 0 게이트**: 개요 `approved:true`. 미승인이면 확장 안 함.
- **Phase 4 계기판**: full_manuscript 에 homiletic_audit — 강사화·재판관화·상투구·논문체 + 드리프트(보이스 이탈)·회피표현. 전체 원고라 정밀. 점등은 비점수 신호(판단은 설교자).
- **Phase 5 sign-off**: 설교자 최종 승인. AI sign-off 금지.

HITL 결정 카드(✅/↻/✏️/⏸)는 모든 Phase 사이.
